# Plateforme Web — v1.0.39

## [FIXED]
- **LEAK-015** — `js/components/decomposition-panel.js` + `js/viewers/volume-viewer.js` : `DecompositionPanel.init()` installait `VolumeViewer.setOnPostRender(...)` **une fois et sans aucun moyen de désenregistrement** — le hook restait branché pour toute la vie de la page et capturait en closure l'état du module (`_canvases` = un `<canvas>` + son contexte 2D par vignette). Couplage `VolumeViewer ↔ DecompositionPanel` non démontable, contraire à la règle 1.2 (teardown d'outil). Correctif :
  - `VolumeViewer.setOnPostRender(cb)` retourne désormais un **unsubscribe idempotent** (`() => { if (_onPostRender === cb) _onPostRender = null; }`).
  - `DecompositionPanel` conserve cet unsubscribe (`_offPostRender`) et expose un `dispose()` qui annule le rendu différé en cours, **désabonne** le hook et **relâche** `_canvases`. `dispose` est câblé sur `pagehide` (le seul vrai événement de teardown d'une page mono-shot — et déclenché par iframe sur la page Compare). Le hook reste vivant entre ouverture/fermeture du panneau (pas de désabonnement dans `_closePanel`, qui casserait la réouverture) ; il n'est libéré qu'au teardown.
- **LEAK-001 / ELE-30 (câblage)** — `VolumeGrid.dispose()` (ajouté en v1.0.29 mais jamais appelé) est désormais **câblé** sur `pagehide` dans `_initVolumeGrid()` (avec garde anti-double-binding `_gridDisposeBound`), libérant les ressources GPU des groupes grille/axes (géométries, materials, `CanvasTexture` des sprites X/Y/Z) au teardown. Le contrat `dispose()` est ainsi réellement honoré au lieu d'être factice.
- Test : `tests/js/test_leak015_teardown.mjs` (structurel : `setOnPostRender` retourne un unsubscribe idempotent ; `_initVolumeGrid` câble `pagehide → VolumeGrid.dispose()` avec garde ; `init` capture l'unsubscribe et câble `pagehide → dispose` ; `_closePanel` ne désabonne PAS ; `dispose()` annule le timer, désabonne et vide `_canvases` ; `dispose` exposé sur l'API) + `node --check`. Non-régression : `test_volume_grid_dispose.mjs` (ELE-30), `test_decomp_debounce.mjs` (ELE-27).
