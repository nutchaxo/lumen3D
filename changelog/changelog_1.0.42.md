# Plateforme Web — v1.0.42

> Lot « intégrité du pipeline de streaming » — `brick-loader.js` + `brick-decode-worker.js`. Le chemin de streaming doit être final et robuste (Rule 1.1) : ces correctifs ferment des défauts de **corruption silencieuse de données scientifiques** et un vecteur de **path traversal** côté manifest.

## [FIXED]
- **SEC-017** (Rule 1.4) — `js/core/brick-loader.js` : une URL de pack issue du manifest (`brickTransport.brickToPack[*].url`) était concaténée telle quelle sur le `basePath` du dataset et fetchée — un manifest hostile avec `url: '../../../../etc/passwd'` (ou une URL absolue / protocole-relative) provoquait un **path traversal** au fetch. `_validateManifest` rejette désormais tout manifest contenant une URL de pack non sûre (segment `..`, scheme `:`, `//host`) **avant montage** (ferme le vecteur pour tous les consommateurs, y compris le worker de fetch) ; garde défensive identique dans `_buildPackIndex`.
- **BUG-065** (Rule 1.4 / 1.1) — `brick-loader.js` + `brick-decode-worker.js` : un `brickPacking` absent ou de mode inconnu retombait sur un défaut `{mode:'vertical'}` → la mosaïque grid 8×8 était lue en **linéaire** → volume **mélangé silencieusement** (`ok:true`). Suppression du défaut vertical (`|| {}`), validation du `brickPacking` à `_validateManifest` (mode ∈ {grid, vertical} ; `webp-lossless` **exige** un grid valide — le préproc en écrit toujours un), et **échec franc** (`ok:false`) du worker sur mode inconnu/absent (le loader surface alors la brick droppée via `onBrickError`, ELE-20) au lieu de monter des voxels corrompus.
- **EDGE-055 / BUG-034** — `brick-loader.js` `bricksForRegion` : les index de brick n'étaient pas bornés → `maxNorm=1.0` émettait un index hors grille (`floor(dim/bs) == n` pour des dims multiples exacts de `bs`) et un `minNorm` négatif émettait des coords négatives. Les appelants compensaient avec `maxNorm=0.9999` + filtre `hasBrick()` ; ajout d'un clamp `[0, ceil(dim/bs)-1]` par axe → fonction correcte seule et robuste à un futur appelant sans ces gardes.
- **LEAK-013** (Rule 1.2) — `brick-loader.js` `clearCache()` ne relâchait jamais le canvas de repli de décodage main-thread (`_fallbackCanvas`/`_fallbackCtx`, dimensionné à la plus grande mosaïque vue, `willReadFrequently`). Mise à `null` au teardown pour le rendre GC-able au switch de dataset.
- **LEAK-011** (Rule 1.2) — `brick-decode-worker.js` : l'`ImageBitmap` décodé (`createImageBitmap`) n'était jamais `.close()` — un handle graphique fuyait à chaque décodage de brick (un par brick 64³ streamée). Libération dans un `finally` sur tous les chemins (succès, suppression-cancel, erreur).

## [OPTIMIZED]
- **PERF-022** — `brick-loader.js` `loadBrickTasks` : le rendement (`yield`) par brick était un `setTimeout(1)` inconditionnel, clampé à ≥ 1 ms (souvent 4 ms+ en onglet throttlé) → plafonnait le débit de chaque worker indépendamment de la vitesse fetch/décodage. Remplacé par un yield à budget temps (~8 ms) partagé, appliqué aussi au replay des bricks en cache (gate de l'ancien `cacheHits % 4`).

## [TESTS]
- `tests/js/test_streaming_integrity.mjs` (nouveau) — harnais Node vm : rejet manifest sur URL de pack `..`/absolue/protocole-relative et acceptation d'une URL relative normale (SEC-017) ; rejet `webp-lossless` sans grid, mode de packing inconnu, `cols` non positif, et acceptation d'un grid valide / raw-u8 sans packing (BUG-065) ; clamp `bricksForRegion` pour `maxNorm=1.0` et `minNorm` négatif (EDGE-055/BUG-034) ; worker `ok:false` sur mode inconnu/absent (BUG-065) ; assertions structurelles `bmp.close()` en `finally` (LEAK-011), `clearCache` nulle le canvas (LEAK-013), suppression du `setTimeout(r,1)` + yield à budget (PERF-022). Non-régression : `test_brick_decode_cols.mjs`, `test_brick_decode_cancel.mjs`, `test_brick_loader_manifest_validate.mjs`, `test_brick_size_64.mjs`.
