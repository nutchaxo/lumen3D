# Plateforme Web — v1.11.0

> **White-label — catalogue de plugins first-party curé & signé (Objectif 4).** Le panneau admin gagne
> un **catalogue de plugins** que l'opérateur parcourt et **installe en un clic**. Entièrement curé et
> first-party (seul l'éditeur y publie), distribué en **releases signées Ed25519**, l'installation est
> **toujours initiée par l'opérateur**, **vérifiée fail-closed**, et atterrit dans le **même trust gate +
> bac à sable** que n'importe quel plugin — **aucune nouvelle surface d'exécution de code arbitraire**.
> Réutilise la machinerie de vérification/extraction/approbation déjà en place. Jumeaux Python **et** PHP.
> Bump de la ligne mineure (Y). Voir [DOCS/whitelabel/PLAN.md](../DOCS/whitelabel/PLAN.md) §5.

## [ADDED]
- **Marketplace serveur — jumeaux Python + PHP** ([dev_server.py](../dev_server.py), [api/_admin_lib.php](../api/_admin_lib.php), [api/admin.php](../api/admin.php)) — nouvelles actions admin (session + CSRF ; install re-authentifié par mot de passe) : `marketplace_catalog` (liste annotée installé/compat), `install_plugin`, `uninstall_plugin`. **Clé de signature dédiée** `_MARKETPLACE_PUBKEY_HEX` / `MARKETPLACE_PUBKEY` **séparée de la clé de release cœur** (autorité de signature des plugins découplée, rotation indépendante) ; **vide ⇒ intégrité sha256 seule + avertissement, posée ⇒ signature OBLIGATOIRE fail-closed**. `_MARKETPLACE_CATALOG_URL` pointe le catalogue signé (sig en `.sig`). **Pipeline d'installation** : catalogue signé vérifié → download zip (plafonné) → **signature du `SHA256SUMS` vérifiée (Ed25519, fail-closed)** → sha256 du zip lu depuis ces octets authentifiés → **extraction durcie** (rejet traversal/absolu/drive/backslash, plafonds entrées+taille) → validation `plugin.json` id/placement → **gate de compatibilité** → move atomique dans `js/modules/<placement>/<id>/` → **approbation opérateur enregistrée** (hash **recalculé côté serveur** sur les octets sur disque — INV-4) → bump `trustEpoch`. Sur toute échec, l'arbre `js/modules` reste **intact**. `uninstall_plugin` : suppression du dossier + révocation de l'approbation + garde « dernier shader ».

Badge « signature vérifiée » vs « non signé ». Clés i18n `mkt.*` + `admin.navMarketplace` en/fr/es (**parité 939 clés**).
- **Builder de release par plugin** ([tools/build_plugin_release.py](../tools/build_plugin_release.py)) — clone de `tools/build_release.py` pour un seul dossier plugin : zip **déterministe** (entrées triées, timestamp 1980, 0644) + `version.json` (`{id,placement,version,files:{relpath:sha256}}`) + `SHA256SUMS` + **`SHA256SUMS.sig`** (signature Ed25519 détachée via `--sign-seed-hex`/`LUMEN_SIGNING_KEY`) + un `catalog-entry.json` prêt à coller. Écrit `SHA256SUMS` en octets bruts (LF) pour que les octets **signés == publiés** (piège de traduction de fin de ligne Windows).

## Notes
- **Vérification** : fonctions Python testées unitairement — liste non configurée, sûreté des chemins (`../etc`, `tools/../../x` → 400 ; inexistant → 404), vérif signature no-op sans clé, **extraction durcie** (bon / imbriqué / entrée traversal rejetée). Jumeau **PHP** testé (php-portable) : mêmes résultats exacts (liste, chemins 400/404, no-op, catalogue/install non configurés) ; `_admin_lib.php`/`admin.php` parsent proprement (l'extension `zip` est gardée par `class_exists`). **Chaîne cryptographique complète prouvée** : `build_plugin_release.py` signe → `ed25519_pure.verify` valide (signature correcte acceptée, `SHA256SUMS` altéré rejeté, mauvaise clé rejetée), zip **déterministe** (rebuild identique). Le panneau admin **démarre** avec les **10 onglets** (dont Catalogue). Le flux d'installation interactif nécessite un marketplace **keyé** + un plugin publié signé (bootstrap TOFU, comme l'updater cœur) — à exercer par l'opérateur.
- **Sécurité — invariants préservés** : fail-closed à chaque étape ; **INV-4** (hash recalculé serveur sur les octets sur disque, jamais celui déclaré) ; **INV-5** (aucun artefact ne porte le trust store ; l'endpoint est l'unique rédacteur d'approbation, re-auth mot de passe) ; catalogue **signé** (la LISTE de quoi installer n'est pas falsifiable, pas seulement chaque release) ; extraction durcie ; move atomique + nettoyage sur échec. Clé publique à **poser en source + commiter** (comme `_RELEASE_PUBKEY_HEX`) pour survivre aux auto-updates.
- **Mise en route (opérateur/éditeur)** : `python tools/gen_signing_key.py` → coller la clé publique dans `_MARKETPLACE_PUBKEY_HEX` (dev_server) **et** `MARKETPLACE_PUBKEY` (`_admin_lib.php`), renseigner `_MARKETPLACE_CATALOG_URL` / `MARKETPLACE_CATALOG_URL` (URL du catalogue signé), publier les plugins avec `tools/build_plugin_release.py` (seed = secret CI). Clé vide + URL vide = marketplace inactif (état par défaut sûr).
