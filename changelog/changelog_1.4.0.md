# Plateforme Web — v1.4.0

> **Refonte complète du panneau d'administration.** L'admin mono-écran (édition de datasets uniquement) devient une véritable application multi-onglets : menu latéral gauche rétractable, routage par hash, thème clair/sombre, langue, et cinq domaines fonctionnels — **Datasets** (édition + masquer/afficher), **Statistiques** (visites / vues / téléchargements, global et par dataset), **Plugins** (activer/désactiver), **Sécurité** (mot de passe durci), **Version & mises à jour** (releases GitHub, 1-clic guardé). UI/UX léchée, responsive (sidebar → drawer < 900 px), construite **sur les tokens `--adm-*` existants** (zéro nouvelle dépendance — graphiques en SVG maison). Les invariants protégés sont préservés : ordre de découverte des plugins (v0.12.45), écritures atomiques `_atomic_write` (RACE-020), jeton de génération de sélection (RACE-006), sync iframe d'aperçu, repli hôte statique/PHP. Serveur de dev → `v0.13.0`.

## [ADDED]
- **Coquille admin multi-onglets** ([admpan.html](../admpan.html), [css/admin-shell.css](../css/admin-shell.css), [js/pages/admin/shell.js](../js/pages/admin/shell.js), [js/pages/admpan.js](../js/pages/admpan.js)) — sidebar gauche rétractable (248 px ↔ 66 px, drawer + overlay sous 900 px), topbar (titre d'onglet, thème sun/moon persistant, sélecteur de langue, déconnexion), routage par hash (`#datasets` `#stats` `#plugins` `#security` `#updates`) avec deep-link et garde « modifications non sauvegardées ». L'éditeur de datasets éprouvé est porté **intact** dans `js/pages/admin/tab-datasets.js` (dirty tracking, calibration d'orientation, sync canaux/exposition par `postMessage`, save → rebuild_catalog). Plomberie partagée (i18n, fetch authentifié + CSRF, toasts) dans `js/pages/admin/shared.js`.
- **Masquer / afficher des datasets** ([dev_server.py](../dev_server.py), [js/pages/admin/tab-datasets.js](../js/pages/admin/tab-datasets.js), [api/datasets.php](../api/datasets.php)) — champ `hidden` dans `metadata.json` ; un dataset masqué est **omis du `catalog.json` public** (donc absent de l'Explorer) mais reste listé et éditable en admin (grisé + badge). Bascule 1-clic via `POST /api/datasets.php?action=set_visibility` (auth + CSRF) qui écrit le flag et régénère le catalogue. Filtre « Masqués » ajouté à la liste.
- **Statistiques d'utilisation** ([dev_server.py](../dev_server.py), [js/pages/admin/tab-stats.js](../js/pages/admin/tab-stats.js), [js/pages/landing.js](../js/pages/landing.js), [js/pages/viewer.js](../js/pages/viewer.js), [api/telemetry.php](../api/telemetry.php)) — compteurs persistés dans `api/stats.json` : visites du site, vues par dataset, téléchargements (global + par dataset + buckets quotidiens). Beacons `navigator.sendBeacon` (visite sur la landing, vue dataset dans le viewer, dédupliqués par session, l'aperçu admin `mode=admin` est exclu) + **interception serveur** des fichiers `DATA_WEB/<type>/<folder>/download/` (chemin fiable, hors range requests). L'onglet rend des cartes résumé, des sparklines SVG (30 j) et un tableau par dataset triable — aucune librairie de graphes ajoutée.
- **Gestion des plugins** ([dev_server.py](../dev_server.py), [js/pages/admin/tab-plugins.js](../js/pages/admin/tab-plugins.js), [api/admin.php](../api/admin.php), [api/plugins.php](../api/plugins.php)) — liste de tous les plugins découverts groupés par placement (tools / channels / shaders) avec interrupteurs. La désactivation est persistée dans `api/disabled-plugins.json` et appliquée **dans la découverte** (`_serve_plugins` + manifeste statique l'excluent) → le viewer ne charge plus le plugin au prochain rechargement, sans casser l'invariant d'ordre. Garde-fou : impossible de désactiver le **dernier mode de rendu (shader)** actif (refus `409 last_shader`, marqué « protégé » dans l'UI).
- **Mot de passe admin durci** ([dev_server.py](../dev_server.py), [admpan.html](../admpan.html), [js/pages/admin/tab-security.js](../js/pages/admin/tab-security.js), [api/auth.php](../api/auth.php), [api/_admin_lib.php](../api/_admin_lib.php)) — le mot de passe vit désormais dans un fichier dédié `api/admin_credential.json` (hash **PBKDF2-HMAC-SHA256 salé, à sens unique** ; aucun secret réversible), jamais servi en HTTP (dossier `api/` bloqué par `_is_forbidden_static`, + `api/.htaccess` pour les hôtes Apache). Si le fichier est **supprimé**, le panneau affiche un écran de **création** au prochain accès ; cette création est **exclusive** (`O_CREAT|O_EXCL` / `fopen('x')`) — elle ne peut **jamais écraser** un mot de passe vivant (course impossible). Le changement de mot de passe exige le mot de passe courant. `--set-password` (opérateur) écrit le nouveau coffre.
- **Version & mise à jour automatique GitHub** ([dev_server.py](../dev_server.py), [js/pages/admin/tab-updates.js](../js/pages/admin/tab-updates.js), [tools/make_release.py](../tools/make_release.py), [api/admin.php](../api/admin.php)) — l'onglet affiche les versions installées (plateforme web = max des `changelog/changelog_*.md`, serveur de dev, préprocessing), interroge `releases/latest` de `nutchaxo/lumen3D`, et propose une **mise à jour 1-clic guardée** : sauvegarde de l'install (en préservant `DATA_WEB/`, identifiants, stats, logs, `.git`) → téléchargement du zipball → extraction par-dessus l'app (chemins protégés exclus) → **redémarrage** par re-exec `os.execv` (sûr sous Windows), avec barre de progression (`update_status`). Helper `tools/make_release.py` pour taguer `vX.Y.Z` et publier la release (amorce le pipeline, aucune release n'existait).
- **Endpoints serveur** ([dev_server.py](../dev_server.py)) — nouveaux : `POST /api/auth.php?action=setup|change_password` (+ `needsSetup` dans `status`), `POST /api/datasets.php?action=set_visibility`, `POST /api/telemetry.php?action=visit|view|download`, `GET|POST /api/admin.php?action=stats|plugins|set_plugin|version|update_check|update_apply|update_status`. **Parité PHP** ([api/admin.php](../api/admin.php), [api/telemetry.php](../api/telemetry.php), [api/auth.php](../api/auth.php), [api/datasets.php](../api/datasets.php), [api/plugins.php](../api/plugins.php), [api/_admin_lib.php](../api/_admin_lib.php)) : même format de hash PBKDF2 (coffre interopérable), même création exclusive ; `update_apply` renvoie `supported:false` sous PHP (pas d'auto-restart fiable hors serveur Python).
- **i18n** ([lang/en.json](../lang/en.json), [lang/fr.json](../lang/fr.json), [lang/es.json](../lang/es.json)) — 88 nouvelles clés `admin.*` (nav, setup/login, visibilité, sécurité, stats, plugins, mises à jour) ajoutées en parité complète sur les trois langues (167 clés admin chacune).

## [FIXED]
- **Comptage des stats sous le mauvais champ** ([dev_server.py](../dev_server.py)) — `_record_event` incrémentait des clés au singulier (`visit`) alors que le schéma utilise le pluriel (`visits`) ; corrigé via un mapping explicite (détecté au test unitaire avant intégration).
- **Capture des globals classiques depuis un module ES** ([js/pages/admin/shared.js](../js/pages/admin/shared.js)) — `'I18n' in globalThis` est faux pour un `const` de script classique (environnement lexical global, pas l'objet global) ; `I18n`/`Utils` étaient donc `null` et `I18n.init()` n'était jamais appelé. Repris via une référence libre gardée par `typeof` (noms locaux distincts pour éviter le TDZ), comme l'ancien `admpan.js`.
- **Plugin « protégé » non recalculé après bascule** ([js/pages/admin/tab-plugins.js](../js/pages/admin/tab-plugins.js)) — l'état « dernier shader » était figé au chargement ; il est désormais recalculé à chaque rendu depuis l'état en mémoire, donc le dernier shader actif se verrouille immédiatement (le garde-fou serveur restait correct dans tous les cas).
- **Script Lucide bloquant** ([admpan.html](../admpan.html)) — la balise CDN Lucide était synchrone ; passée en `async` + `onload` pour qu'un CDN lent/bloqué ne retarde jamais le chargement de la page.
