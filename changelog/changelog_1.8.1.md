# Plateforme Web — v1.8.1

> **White-label — découplage du moteur (Objectif 1, phase de rollout).** Suite directe du socle
> [v1.8.0](changelog_1.8.0.md) : le mécanisme de configuration d'instance (prouvé sur `index.html`)
> est **déployé sur toutes les pages**, les dictionnaires i18n sont **généralisés** (plus aucun nom
> métier en dur), et les **hypothèses biologiques** restantes du JS cœur passent en configuration.
> La plateforme livrée par défaut ne présume désormais d'**aucun domaine** ; l'identité IRIBHM est
> entièrement portée par `config/instance.json`. Voir [DOCS/whitelabel/PLAN.md](../DOCS/whitelabel/PLAN.md) §2.

## [ADDED]
- **Terminologie d'échantillon localisable** ([js/core/instance-config.js](../js/core/instance-config.js), [config/instance.json](../config/instance.json)) — `specimen.singular/plural` accepte soit une chaîne unique, soit un objet **par locale** (`{ "en":"embryo", "fr":"embryon", "es":"embrión" }`). `InstanceConfig.tokens()` résout la langue courante (`I18n.getLanguage()`) → `en` → première dispo, de sorte que `{specimen}`/`{specimenPlural}` restent **grammaticalement corrects** dans chaque langue. L'instance IRIBHM déclare embryon/embryons localisés ; le défaut neutre reste `sample/samples`.
- **Presets couleur de canaux en configuration** ([config/instance.json](../config/instance.json), [config/defaults/neutral/instance.json](../config/defaults/neutral/instance.json)) — nouveau tableau `channelColorPresets: [{match, color}]`. L'instance IRIBHM reproduit à l'identique l'ancien mapping fluorophore→couleur (gfp→vert, dapi/hoechst→bleu, pecam→magenta, rfp/mcherry/alexa→rouge) ; le défaut neutre est vide (cycle `DEFAULT_COLORS`).

## [OPTIMIZED]
- **Rollout white-label sur toutes les pages** ([explorer.html](../explorer.html), [about.html](../about.html), [compare.html](../compare.html), [tracking.html](../tracking.html), [viewer.html](../viewer.html), [admpan.html](../admpan.html) + contrôleurs [explorer.js](../js/pages/explorer.js)/[about.js](../js/pages/about.js)/[compare.js](../js/pages/compare.js)/[tracking.js](../js/pages/tracking.js)/[viewer.js](../js/pages/viewer.js)/[shell.js](../js/pages/admin/shell.js)) — chaque page applique le motif de référence : `<title>`/`<meta>` via placeholders serveur `{{SITE:pageTitles.<page>|…}}`, marque/organisation/pied de page via `data-instance`, `<body data-page="…">`, chargement `InstanceConfig.load()` **avant** `I18n.init()` puis `applyHead()`+`applyDom()`. Le panneau admin (ESM) charge `InstanceConfig` dans `shell.boot()` (binding global-lexical comme `I18n`/`Utils`). Les titres dynamiques par dataset (viewer, tracking) utilisent `InstanceConfig.get('brand.name')`.
- **i18n généralisé — plus aucun nom métier en dur** ([lang/en.json](../lang/en.json), [lang/fr.json](../lang/fr.json), [lang/es.json](../lang/es.json)) — les valeurs des clés porteuses de domaine (mouse-embryo / IRIBHM / ULB / Migeotte / DAPI-GFP-Pecam1 / « Imaris-like » / « Anatomical ») sont réécrites en **tokens** (`{brand}`, `{org}`, `{tagline}`, `{specimen}`, `{specimenPlural}`, `{Specimen}`, `{SpecimenPlural}`) ou en formulation neutre, dans les **trois** langues en lockstep. **Parité vérifiée : 740 clés identiques en/fr/es, JSON valide, zéro littéral de domaine résiduel hors `plugins.*`** (seule rétention volontaire : `.imaris_track` comme identifiant de format de fichier supporté, pas une marque). Aucune clé ajoutée/supprimée/renommée — valeurs seules.
- **Labels de relation dé-domanisés** ([js/core/catalog.js](../js/core/catalog.js)) — `_relationLabel`/`_relationDescription` utilisent le nom d'échantillon configuré (`Same {specimen}/stage`, `Matched by {specimen} and stage metadata`) au lieu de « embryo » codé en dur (la clé de type `same-embryo` reste inchangée — elle n'est émise que pour des datasets portant des métadonnées échantillon+stage).
- **`_colorForChannel` sans présomption fluorescence** ([js/components/channel-panel.js](../js/components/channel-panel.js)) — la table fluorophore→couleur en dur est remplacée par une lecture de `InstanceConfig.channelColorPresets` (sous-chaîne→couleur), repli sur le cycle neutre `DEFAULT_COLORS`.

## [FIXED]
- **Double-source des titres — supprimée sur toutes les pages** ([about.js](../js/pages/about.js), [explorer.js](../js/pages/explorer.js), [compare.js](../js/pages/compare.js), [tracking.js](../js/pages/tracking.js), [viewer.js](../js/pages/viewer.js)) — les `document.title = '…IRIBHM…'` en dur qui écrasaient le titre injecté par le serveur sont retirés (ou, pour les titres dynamiques par dataset, recomposés depuis `InstanceConfig`), conformément à PLAN §1.3.
